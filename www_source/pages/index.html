Scalable frontend architecture

HTML-first

Write modern HTML and add interaction using the data-viewmodel attribute. The correct JavaScript files are automatically loaded and applied to the correct elements.

  • JavaScript files will be loaded lazily when needed
  • An element is controlled by one JavaScript file only
  • Don't worry about loading scripts in the wrong order

Two-way bindings

The JavaScript file is bound to the HTML with data-bind using KnockoutJS. These bindings automatically update the DOM and the JavaScript when either one changes.

  • The HTML tells you exactly what it does and only what it does
  • The view updates when it needs to, automatically
  • The only thing an element can do is what it says it can do
  • page.html
  • FirstVM.js

<div data-viewmodel="FirstVM">
  <label>First name</label><br>
  <input data-bind="value: firstName, valueUpdate:'afterkeydown'"><br>
  <label>Last name</label><br>
  <input data-bind="value: lastName, valueUpdate:'afterkeydown'"><br>
  <span data-bind="text: fullName"></span>
</div>
        

define(['knockout'], function(ko){
  return function FirstVM(){
    this.firstName = ko.observable("");
    this.lastName = ko.observable("");
    this.fullName = ko.computed(function(){
      return this.firstName() + " " + this.lastName();
    }, this);
  };
});

Long Living Pages

Add interactivity to your existing pages. A single main.js file lets you easily add functionality to multiple pages, without creating an entire Single Page application from scratch for every page

  • Only one main.js file to bootstrap all your pages
  • Add multiple interactive areas to a single page
  • Reuse modules across pages

Multi-page application

DecoJS lets you work with both a frontend and server-side framework when creating your application. Continue to use the router, controllers and view-engine on the server, and add frontend functionality only where you need it.

  • Create single page applications without a frontend router
  • Use server-side controllers and views without conflicting with the frontend
  • Create applications with multiple pages, not a single large page
